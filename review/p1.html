チャット型 vs Claude Code

「答えてくれるAI」から「自分でやってくれるAI」へ

確認クイズ

Q 1
スライドでは、チャット型AIを「タイミーさん」、Claude Codeを「社員さん」にたとえていました。この対比が表しているClaude Codeの特徴はどれですか？
解説スライド「THE DIFFERENCE」では、チャット型＝「タイミーさん」（毎回ゼロから説明が必要・答えるだけ・完結させるのは自分）、Claude Code＝「社員さん」（会社のことを知ってる・やりきってくれる・育てれば育つほど賢くなる）と対比していました。Bが社員さん側の特徴です。
Q 2
FEATURE 01「やりきってくれる」では、Claude Codeが「答えるまで」から何へ進化したと説明していましたか？
解説スライド「FEATURE 01」のキーワードは「答えるまで」→「完結まで」です。コピペ・保存・整理まで全部自動。返ってきた答えを自分でファイルに貼る、その作業がなくなる——これが「やりきってくれる」の意味です。
Q 3
FEATURE 02「自分仕様にできる」で紹介された、Claude Codeに渡す「説明書」のファイルは何という名前でしたか？
解説スライド「FEATURE 02」では、「私はSNS運用者で、この形式で出して」という説明書（CLAUDE.md）を渡せると紹介されました。これで「毎回ゼロから説明」が「最初から知ってる状態」に変わります。CLAUDE.mdの詳しい使い方は第4回で扱われます。
Q 4
FEATURE 03「使うほど賢くなる」では、チャット型との違いをどう表現していましたか？
解説スライド「FEATURE 03」では、好み・業務パターン・言い回しが蓄積されていくと説明。「チャット型は毎回リセット。Claude Codeは育てられる」——「毎回リセット」→「育てれば育つほど」という対比が示されていました。
Q 5
PRICING／ROIのスライドで示された、月3,000円が「元が取れる」かどうかの判断式はどれですか？
解説スライド「PRICING」の数式は「あなたの時給 × 消えた作業時間 ＞ 3,000円」で「元が取れる」。さらに「ROI」スライドでは、それ以上削減できれば「プラスの投資」になり、新しい収入源をつくれば回収→投資→収益とさらに広がる、と続きます。
